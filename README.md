## 三轨预测法（九维度基本面 × 市场 × 爆冷）
每场跑**两条独立预测轨**，并列展示、不互相覆盖；**一致=高置信，分歧=重点信号**。

**轨一·九维度基本面**（自下而上）：派独立 Agent，各只负责一个维度（互不污染），输出一个**比分**预测：

| # | 维度 | 看什么 |
|---|------|--------|
| 1 | 首发阵容核验 | 身价/俱乐部级别/位置短板 |
| 2 | 伤停情况 | 核心缺阵权重 vs 轮换 |
| 3 | 历史赛事底蕴 | 世界杯履历/大赛基因/球员大赛经验 |
| 4 | 历史交锋 H2H | 交锋史走势（样本不足则降权） |
| 5 | 最近比赛状态 | 近 8-10 场战绩 + 对手强度加权 + 状态曲线 |
| 6 | 裁判尺度 | 主裁牌量/点球频率/严松/对抗容忍度 |
| 7 | 气温环境 | 当日实测天气 + 球队气候适应性（**看实况非城市标签**）|
| 8 | 教练战术 | 体系相生相克/临场调整口碑 |
| 9 | 战略卡位（出线/淘汰赛策略） | ① **末轮**:图第二名避强敌、已出线轮换、死亡组放水→不全力争胜(需算积分+对阵树)。② **首轮也有关系(48队赛制,Osk 6/16 修正)**:8个最佳第三名出线→**净胜球是生死线**,强队打弱旅有动机刷大胜攒净胜(↑净胜/大球);"首败70%出局"→两强直接对话双方更谨慎不愿先输(↓净胜)。**不再默认中性**;仅在确无策略含义时补"中性" |

**轨二·赔率市场预测**（自上而下，看赔率**变化**+庄家博弈，输出一个**赢家**预测）：
1. **去水(de-vig)** — 1X2 隐含概率归一到 100%，剔除抽水，得市场公平概率
2. **开盘→现盘移动** — 赔率朝谁收窄=资金流入；幅度=信号强度
3. **庄家博弈** — 利锐盘(Pinnacle，聪明钱)vs 软庄(bet365/FanDuel，公众盘)分歧；**逆向赔率移动 RLM**（公众押 A 但线朝 B 收窄=聪明钱在 B，最强信号）；诱盘警惕
4. 数据现实：去水+移动稳定可得；公众%/利锐盘历史移动常抓不到 → **拿不到标"数据不足"，绝不硬编**
5. 输出：市场独立 winner + 信心 + 移动/聪明钱注释

**每条轨各输出三类预测**：① 猜比分(全场) ② 猜胜负(半场+全场,胜/平/负) ③ 猜进球数(总进球+大小球2.5)。详情页用「3类×2轨」对照矩阵呈现,差异行高亮。
- 九维轨:比分=pred；半场比分=ht8；胜负/进球数由比分推导。
- 赔率轨:比分=os(市场推导)；半场胜负=oht；全场胜负=odds.winner；进球数=oou(大小球)。

**轨三·爆冷预测**（独立第三轨，建模"favorite 失手"的尾部场景）：综合伤停/气温/交锋/状态/教练等利空 favorite 的维度 + 市场不败概率，输出**最可能的冷门结果**：① 爆冷比分 ② 爆冷胜平负 ③ 爆冷大小球 + **爆冷概率%**(= favorite 不赢概率) + 一句话理由。数据字段 `upset:{pred,winner,ou,prob,why}`。半场不适用。高概率(≥40%)即重点预警。**爆冷比分四原型(强制按场景定形,严禁默认 1-1——Osk 6/16+6/19 两次指出的偷懒)**：每写一张爆冷牌先打标签,再选比分,不许塌缩成"1-1·小":
- 🪨 **真鱼腩摆大巴**(代差悬殊+弱旅无火力,如海地/佛得角)→ **0-0 / 1-1 平,under**;弱旅 R1 零进球→优先 **0-0**。
- ⚡ **弱旅有火力/快反**(有真攻击点,如巴拉圭/塞内加尔/2022沙特)→ **弱旅偷赢 1-0 / 2-1**,`winner`=弱旅 code,可 over。
- ⚖️ **势均/两强对话**(favorite 仅微热)→ **1-1 / 2-2 平**,双方都攻则 over(如英-克实为 4-2)。
- 🔥 **favorite 被迫追分**(落后情境)→ **2-1 / 3-2 高比分,over**。
- 自检:一批 4 场若爆冷牌**全是平局或全是 under**,就是偷懒信号,回去按原型重标。历史正常分布≈平局多数但**应含 1-2 张弱旅偷赢 / over**。

**爆冷气候（研究支撑的阶段因子,非"连冷传染")**：经查证(FIFA技术组"70%首败出局"+Gracenote"2022小组赛12次冷门破纪录"+足球低事件数连冷与随机不可区分),爆冷有**真实阶段规律**但**无场次间传染**:
- ✅ **小组首轮**爆冷率系统性偏高(强弱都保守摆大巴+强队生疏+低比分放大方差)→ 首轮 `upset.prob` 基线上调(6/14-6/16 首轮场次的高爆冷概率即由此)。
- ✅ **末轮死局**:强队已出线轮换→对其实力打折(由战略卡位维度处理);**淘汰赛**:加时/点球(~31%)把favorite压向50%。
- 🟡 **届级爆冷气候**:顶栏指示器实时算"本届热门翻车率"(已赛场次中博彩热门未取胜占比),作**背景参考/弱先验**(≤±2pp),不强加权。
- ❌ **红线(丢弃)**:"昨天/本届已连爆N场→今天更易爆"的连冷加成——这是 clustering illusion(认知错觉),严禁当强信号。
- **真因提醒**:6/14-6/15"强队集体翻车"的根因是**小组首轮**(阶段),不是"冷传染冷"。

**两轨对比**：winner 一致→✓置信叠加；不一致→⚡重点关注（如巴西-摩洛哥：九维看平、市场看巴）。三轨在详情页「三轨预测对照」矩阵并列(① 九维 / ② 市场 / ③ 爆冷)。

## 历届世界杯参照系（届性 · 爆冷推演依据）
> 把"历届比分/爆冷演化规律"建成参照系,叠加本届早期定性,用于推演后续常规比分 + 爆冷比分。数据基于 Statista 22 届(去训练记忆)。

**A. 进球演化**：历史均值 **2.83/场**(964 场 2720 球)。三时代——① 早期高产(1930-58,3.9-5.4,峰值 1954=5.38)② 防守低谷(1966-90,谷底 1990=**2.21**)③ 现代回升(1990-2022 稳定 **2.5-2.7**:94=2.71/98=2.67/02=2.52/06=2.30/10=2.27/14=2.67/18=2.64/22=2.69)。**现代正常区间≈2.6-2.7**,判断某届"高产/低产"以此为锚。

**B. 爆冷演化**：无干净逐届率,但质性史清晰——**2002=现代最爆冷届**(卫冕法国小组出局且零进球+阿根廷出局+塞内加尔揭幕爆冷+韩/土四强);**2018** 德国(卫冕)小组垫底。**"卫冕魔咒"**:近 5 届 4 届卫冕冠军小组出局(法 02/意 10/西 14/德 18)。**长期趋势:扩军+平民化→爆冷走高**。48 队(本届)+8 个最佳第三名机制→弱旅"搏平也能出线"→摆大巴动机历史最强→平局/爆冷的结构支撑**比往届更硬**。

**C. 如何给本届定型(早期信号读法)**：开赛 ~20 场后看三个量——① 场均进球 vs 2.6-2.7 基线;② 平局率 vs ~27% 常态;③ 热门翻车率 vs ~35-40% 首轮常态。三项齐高=**2002 型高爆冷届**。
> **本届(截至 Day7/28 场)判定**：场均 **3.18**(高,但被德7-1/瑞5-1/英4-2/加6-0 屠杀拉高,中位仍低比分→**两极分化**)、平局率 **35.7%**、热门翻车 **46%**→**坐实"2002 型高爆冷届"**;且**无连冷传染**(实证:Day4 H/G 组全平、Day5 I/J 组大热全胜,按组/阶段成簇)。

**D. 推演规则(历届×本届合成,本届适用)**：
1. **进球分场型**：代差大(去水>75%)→碾压方屠杀 **3-5 球**(别压缩=头号校准债);势均(去水<60%)→低比分/平局。
2. **爆冷(核心)**：本届=2002 型 → favorite 非碾压(去水<65%)且有变量(铁桶/环境/伤停/卡位)→爆冷概率 **+5pp**;弱旅可"搏平出线"→爆冷形态偏**逼平**非爆胜(除非弱旅有真火力)。
3. **守红线**：真碾压(去水>80% 或代差悬殊)**不抬爆冷** + **不连冷加成**。
4. **阶段修正**：2/3 轮卡位主导——领先队可能轮换/放水(实力打折)、落后队拼命(更开放)、必争组(全平)更易出冷。
5. 顶栏"冷门气候"指示器实时量化本届翻车率,作弱背景先验(非强加权)。

## 校准教训（持续更新，来自赛后复盘）
1. **全部同向的强势方，不要压缩比分。** 美国 vs 巴拉圭全维度看多→实际 4-1（我只给 2-1）。当信号一致 + 对手防线弱 + favorite 锋线火热，要敢给 3-1/4-1 天花板。
2. **铁血低位防守 + 对手缺终结力 = 平局机器。** 加拿大 vs 波黑→1-1（我主预测 1-0，次选 1-1 命中）。
3. **看天气实况不看城市标签。** "高温城市"当日可能反常湿冷（如福克斯堡 8°C 冷雨）。
4. **战术范式别套旧记忆。** 摩洛哥已非 2022 Regragui 低位铁桶，新帅 Ouahbi 是 U20 冠军进攻型——范式套用前先核实主帅。
5. **两轨分歧是信号不是噪音。** 巴西 vs 摩洛哥九维看平、赔率轨看巴西胜——不强行合并，并列呈现让分歧本身成为"重点关注"标记。注意赔率轨要分辨"聪明钱"还是"公众追名气"（巴西带重伤盘口反收窄=公众推动，去水仅 58% 非碾压）。

## 每30分钟自动刷新赔率（混合方案 C）
市场轨数据已外置为服务器上的 `odds.json`，页面每 90s 自动拉取并覆盖内嵌 `PRED.odds`/`PRED.x`（改 odds.json **零重新部署**，已验证）。
- **结构**：`odds.json` = `{ "_updated":"...", "队1|队2":{winner,prob,conf,move,sharp,os,oht,oou} }`。
- ✅ **机械层(已上线)**：阿里云 cron `*/30 * * * *` 跑 `/root/wc-odds/update-odds.js`(Node) → 调 **API-Football**(免费100/天, header `x-apisports-key`, base `https://v3.football.api-sports.io`, **league=1 season=2026**；注意 league+season 直查 fixtures 返0，需按 `?date=` 查) → 取 **Pinnacle**(利锐盘,无则Bet365) 1X2 去水 + Exact Score(最小赔率)→比分 + Goals O/U 2.5→大小球 + 与 `snap.json` 对比算移动 → 覆盖 odds.json 的 winner/prob/conf/move/os/oht/oou。**只在每场开赛前3h窗口抓(非赛时段0 API调用)**,内嵌全赛程判断窗口,配额安全(~2-5次/run)。
  - key 存服务器 `/root/wc-odds/.env`(chmod600,**不进git/不进网页**)。脚本副本在 `server/update-odds.js`。日志 `/root/wc-odds/cron.log`,fixture ID缓存 `fixids.json`。
  - **关键**：脚本只在 `!e.sharp` 时写 sharp，**绝不覆盖 LLM 写的 sharp** → 机械层与LLM层共存。
- **LLM深度层**：在 Osk 的 Claude 会话(本机)按需触发"刷新赔率" → 我重跑赔率agent → 覆盖 odds.json 的 sharp(聪明钱/RLM/诱盘叙事)。不需常开服务器跑 LLM。
- **九维轨**：保持内嵌，不高频刷（基本面一天变一两次，首发/伤停官宣时触发一次）。
- ✅ 第①(odds.json外置+页面自动读)+ ②(cron机械层) 均已上线。③ LLM深度层随时可手动触发。

## 日常工作流
口令：**"预测今天的"** 或 **"继续世界杯预测"**
1. 从 `SCHEDULE.md` 取当天（北京时间）全部场次
2. 每场跑三轨预测独立 Agent + 1 个增量数据 Agent（拉本届已完赛结果，更新积分/状态）
3. 主裁综合 → 输出比分 + 胜平负概率 + 信心等级
4. 赛后我报实际比分 → 回填 `PREDICTIONS.md` + 更新校准教训
